rag-core is an embeddable Python retrieval engine for RAG. Point it at a folder,
ask a question, and get back prompt-ready context with citations. It runs locally
with no API key, and the same calls scale to a configured vector store and hosted
embeddings when your application needs them.

It owns the retrieval path: ingest, chunking, indexing, search, context assembly,
traces, and eval hooks. Your application keeps authentication, tenancy,
connectors, chat, and generation. rag-core stays a retrieval core, not a hosted
service.

## Install

```bash
pip install rag-core   # or: uv add rag-core
```

## Index a folder

```python
import rag_core

idx = rag_core.index("./docs")
print(idx.context("How can invoices be paid?"))
```

`index()` embeds the folder with a local model (FastEmbed, no API key) and returns
a handle. `context()` returns prompt-ready context with citations. `search()`
returns ranked hits when you want to build the prompt yourself.

<Callout type="info">
  The first run downloads the local embedding model. Later runs reuse the cached model.
</Callout>
